①
전체 키워드 기사 수집 + 빈도 집계
10개 키워드를 순회하며 지정 기간 내 기사를 모두 수집합니다.
②
키워드 매칭 빈도 = 랭킹 지표
Naver News API는 조회수를 직접 제공하지 않습니다. 더 많은 키워드에서 반복 노출된 기사일수록 중요도가 높은 기사로 간주합니다.

02
개선방안
📰
오늘의 마케팅 이슈
랭킹 산정 기준 개선
현재 문제점
Naver News API는 조회수를 직접 제공하지 않아 우선순위 산정 기준이 모호합니다.
현재는 복수 키워드에서 반복 노출된 기사를 높은 중요도로 간주하는 방식을 사용하지만, 이는 실제 독자 관심도와 괴리가 있을 수 있습니다.
개선 방향
언론사 신뢰도·규모를 가중치로 반영해 주요 언론사 기사를 우선 노출
🟢
플랫폼 트렌드 — 네이버
API 의존성 및 데이터 완성도 개선
현재 문제점
Naver DataLab API에 전적으로 의존하는 구조입니다.
개선 방향
추가 검토 필요(자체 API 이상의 데이터 소스를 구하는 것이 쉽지 않음)
▶
플랫폼 트렌드 — 유튜브
광대한 데이터 필터링 및 마케팅 특화
현재 문제점
YouTube 인기 급상승 데이터는 엔터테인먼트·게임·스포츠 등 마케팅과 무관한 콘텐츠를 대거 포함합니다.
카테고리 필터 없이 수집하면 마케터에게 유용한 인사이트보다 단순 인기 콘텐츠 목록에 그칠 수 있습니다.
개선 방향
YouTube API 카테고리 필터(마케팅·비즈니스·교육 등)를 적용해 마케팅 관련 콘텐츠만 선별
인기 급상승 순위 대신 Hype 지수를 분석해 폭발적으로 증가 중인 콘텐츠를 포착
영상 제목·설명에 Claude를 활용한 마케팅 연관성 스코어링으로 노이즈 제거
📱
플랫폼 트렌드 — 인스타그램 · 틱톡 · X
실시간성 한계 및 비용 문제 해결
현재 문제점
SNS 트렌드는 시간별·분 단위로 계속 바뀌기 때문에 현재의 1회성 수집 방식은 실시간 트렌드를 정확히 반영하기 어렵습니다.
공식 API가 없어 뉴스 기사를 간접 수집하는 방식이라 지연이 불가피하며, 실시간 모니터링을 추가할 경우 비용이 크게 증가합니다.
개선 방향
지속적인 모니터링을 통해 시간대별 트렌드 변화 추적. 단 이 경우 모니터링 비용을 고려해야 함
트렌드 지속 시간을 측정해 일시적 이슈 vs 지속 트렌드를 구분하는 태깅 시스템 도입
🏢
업종별 마케팅 인사이트
품질 고도화 및 개인화
현재 상태
현재 11개 업종에 대해 Claude가 성공사례·이슈·트렌드 3종 리포트를 생성하는 구조입니다. 업종별 추가 개선 방향은 지속 검토 중입니다.
개선 방향
업종별로전문 프롬프트를 최적화하여 업종 특성에 맞는 더 정교한 분석 생성
Claude 분석에 경쟁사 사례·벤치마크 데이터를 함께 제공해 구체성 향상
