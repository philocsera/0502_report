Section 01

분석 파이프라인

URL을 입력하면 아래 순서로 분석이 진행됩니다. 전체 소요 시간은 약 1분입니다.

1

URL 정규화 및 크롤링

입력된 URL에 https:// 프로토콜을 붙이고, Cheerio 라이브러리로 HTML을 파싱합니다. 제목·메타태그·헤딩·이미지·링크·구조화 데이터 등 SEO 관련 신호를 추출합니다.

2

기술 SEO 체크리스트 생성

추출한 데이터를 기준값과 비교하여 각 항목을 통과 / 경고 / 실패로 분류합니다. 실패 항목에는 구체적인 개선 제안이 첨부됩니다.

3

브랜드 인지도 점수 산출

SNS 채널 연결, About 페이지, 로고, 연락처 존재 여부를 탐지하여 100점 만점으로 점수화합니다.

4

외부 API 병렬 조회

DART 공시 API로 기업 규모(직원 수·매출)를 조회하고, Chrome UX Report(CrUX) API로 실제 사용자 경험 지표(LCP·FCP·INP·CLS·TTFB)를 수집합니다. 두 API는 병렬로 호출됩니다.

5

Claude AI 분석

크롤링된 본문과 메타데이터를 Claude Sonnet 모델에 전달합니다. 업종·기업 규모·타깃 고객·브랜드 스토리·차별화 포인트·마케팅 전략·개선 제안·경쟁사 분석을 JSON 형식으로 반환받습니다. AI 응답 수신 후 경쟁사 URL에 HEAD 요청(3초 타임아웃)을 병렬로 보내 실제 접근 가능 여부를 검증합니다. 404·타임아웃 URL은 자동으로 제거됩니다.

6

매출 추정 (선택)

DART 데이터가 있으면 실제 신고 매출을 사용하고, 없으면 직원 수 벤치마크 또는 본문 내 매출 언급 문구를 파싱하여 추정 매출 범위를 제공합니다.

결과 캐싱 & 히스토리 — 분석 결과는 localStorage에 최대 10건, 24시간 동안 보관됩니다. 동일 URL 재입력 시 "N시간 전 분석 결과" 힌트가 표시되어 API 재호출 없이 즉시 결과를 볼 수 있습니다. 메인 페이지 하단 최근 분석 기록에서 이전 결과를 조회·삭제할 수 있습니다.
Section 02

기술 SEO 체크리스트

HTML에서 직접 추출한 데이터를 아래 기준으로 평가합니다. 각 항목은 통과경고실패 중 하나로 분류됩니다.

항목 판단 기준 근거
페이지 제목 (Title) 5자 이상, 32자 이하 한국어 기준 Google SERP 절단 지점(~32자). 하한은 키워드 부재 방지 최솟값.
Meta Description 1자 이상, 80자 이하 한국어 SERP 스니펫 표시 가능 범위(~80자). 영문 기준 160자와 달리 한국어는 절반 수준.
H1 태그 페이지당 정확히 1개 H1은 페이지의 주제를 선언하는 역할. 0개면 주제 미선언, 2개 이상이면 검색엔진에 혼란을 줄 수 있음.
HTTPS 보안 URL이 https://로 시작 Google은 2014년부터 HTTPS를 랭킹 신호로 반영. 미적용 시 브라우저 경고 표시.
Viewport 메타 태그 존재 시 통과, user-scalable=no 포함 시 경고 Google 모바일 퍼스트 인덱싱 필수 항목. user-scalable=no는 사용자 확대 기능을 차단해 WCAG 접근성 지침(1.4.4) 위반에 해당.
Canonical 태그 <link rel="canonical"> 존재 여부 파라미터 URL 중복 색인 방지. 자기 참조 Canonical도 권장 관행.
Open Graph 태그 og:title과 og:description 모두 존재 SNS 공유 시 제목·설명·이미지 자동 표시. 둘 중 하나라도 없으면 경고.
이미지 Alt 텍스트 80% 이상 충족 시 통과, 60% 이상 경고 Alt 미입력 시 이미지 검색 노출 불가. 시각장애인 접근성(WCAG) 기준과 동일.
robots.txt /robots.txt 200 응답 여부 크롤러에게 허용/차단 경로를 안내. 없으면 크롤러가 전체 사이트를 임의로 탐색.
Sitemap.xml /sitemap.xml 200 응답 여부 검색엔진이 모든 페이지를 발견할 수 있도록 안내. Google Search Console 등록과 연계.
구조화 데이터 (Schema.org) JSON-LD 형식 <script type="application/ld+json"> 존재 Google 리치 스니펫(별점·FAQ·이벤트 등) 노출 조건. Organization/LocalBusiness 타입 권장.
본문 콘텐츠 양 300단어 이상 통과, 150단어 이상 경고 Google의 '얇은 콘텐츠(thin content)' 패널티 회피 최솟값. 품질 콘텐츠 기준의 하한선.
헤딩 구조 (H2~H3) H2 태그 2개 이상 콘텐츠 구조 명시로 검색엔진 이해도 향상. 사용자 가독성 개선 효과도 있음.
내부 링크 10개 이상 통과, 3개 이상 경고 내부 링크는 크롤러의 사이트 구조 파악 및 페이지 간 링크 권한(PageRank) 분산에 활용. 과소 연결 시 고아 페이지 발생 위험.
Nofollow 링크 비율 외부 링크 중 nofollow 비율 50% 미만 통과, 80% 미만 경고. 외부 링크 없으면 항목 미표시. 신뢰할 수 있는 외부 사이트에도 nofollow를 일괄 적용하면 자연스러운 링크 생태계 형성이 어려움. 스팸·광고성 링크에만 nofollow 적용 권장.
개선 제안 코드 예시 — 실패·경고 항목에는 바로 적용할 수 있는 코드 예시가 함께 제공됩니다. Canonical 태그, robots.txt, sitemap.xml, OG 태그, 구조화 데이터 (JSON-LD), 이미지 Alt 텍스트, Viewport 태그 항목에서 확인할 수 있습니다.

점수 산출

각 항목의 통과·경고·실패를 각각 100·50·0점으로 환산하여 전체 평균을 기술 SEO 점수(0~100)로 사용합니다.

항목별 탐지 방법

  • 📱
    SNS 채널 연결 — 대상 플랫폼: Instagram, YouTube, Twitter/X, 네이버블로그, 네이버포스트 (5개).
    HTML 내 <a href>에서 각 플랫폼 URL 패턴을 탐지합니다. 플랫폼당 10점, 3개 이상은 상한(30점) 고정.
  • 🏢
    About / 소개 페이지
    링크 URL에 /about, /company, /introduce 등이 포함되거나, 링크 텍스트에 "회사소개", "브랜드스토리" 등이 포함된 경우 탐지.
  • 🖼️
    로고 이미지
    ① img src/alt/class/id에 "logo" 또는 "로고" 포함 ② 이미지가 class*="logo" 부모 요소 내에 위치 ③ CSS 배경 이미지 방식(<div class="logo">) ④ og:image 존재 — 네 가지 방법 중 하나라도 해당되면 탐지.
  • 📞
    연락처 정보
    본문 텍스트에서 전화번호 패턴(01X-XXXX-XXXX, 지역번호 형식), 이메일 주소, 또는 시·도 지역명 중 하나라도 발견되면 탐지.
Section 04

AI 브랜드 분석

크롤링된 본문·메타데이터를 Claude Sonnet 모델에 전달하여 아래 항목을 JSON으로 반환받습니다.

추정 업종 기업 규모 타깃 고객 브랜드 스토리 차별화 포인트 마케팅 전략 우선순위별 개선 제안 경쟁사 분석 경쟁 공백·기회 영역 메시지 일관성 타깃 명확성 차별화 강도
개선 제안 우선순위 — AI는 각 개선 제안을 critical(긴급), warning(주의), info(참고) 세 단계로 분류합니다. 매출·트래픽에 직접 영향을 미치는 항목이 긴급으로 분류됩니다.
경쟁사 URL 자동 검증 — AI가 제안한 경쟁사 URL에 HEAD 요청(타임아웃 3초)을 병렬로 보냅니다. 404 오류나 접근 불가 URL은 결과에서 자동 제거되어 존재하지 않는 경쟁사가 표시되는 문제를 방지합니다.
파싱 안정성 — AI 응답이 도중에 잘리거나 JSON 형식이 깨진 경우, sanitizeJson → repairTruncatedJson 3단계 복구 로직이 자동으로 실행됩니다. 복구 실패 시 원문이 /tmp/ai_parse_error.txt에 기록됩니다.

Section 06

한계 및 주의사항

  • ⚠️
    봇 차단 사이트
    Cloudflare 등 봇 방어가 적용된 사이트는 크롤링에 실패할 수 있으며, 이 경우 분석이 제한됩니다.
  • ⚠️
    단일 페이지 분석
    입력한 URL의 단일 페이지만 분석합니다. 사이트 전체 구조, 내부 링크 깊이, 다른 페이지의 SEO 상태는 반영되지 않습니다.
  • ⚠️
    브랜드 인지도 탐지 정확도
    CSS 배경 이미지 로고, JavaScript로 동적으로 삽입된 SNS 링크 등은 탐지되지 않을 수 있습니다. HTML 정적 마크업 기준으로만 판단합니다.
  • ⚠️
    AI 분석의 한계
    Claude AI의 분석은 크롤링된 텍스트를 기반으로 한 추정입니다. 실제 비즈니스 맥락, 내부 데이터, 오프라인 현황은 반영되지 않습니다.
  • ⚠️
    CrUX 데이터 부재
    트래픽이 낮은 사이트는 Chrome UX Report에 데이터가 없어 실제 사용자 경험 지표(LCP·FCP 등)가 표시되지 않을 수 있습니다.
  • ⚠️
    DART 미등록 기업
    개인사업자 또는 DART에 공시 이력이 없는 법인은 기업 규모 조회가 불가하여 AI 추정값을 사용합니다.
  • ⚠️
    요청 제한 (Rate Limiting)
    동일 IP에서 분당 3회, 시간당 10회를 초과하면 429 오류와 함께 재시도 가능 시간이 안내됩니다. 인메모리 방식이므로 서버 재기동 시 카운터가 초기화됩니다. 프로덕션 환경에서 완전한 방어가 필요하다면 Redis 기반 전환을 권장합니다.
